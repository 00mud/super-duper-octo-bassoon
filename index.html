Theoretical Foundation & Background

The Latency of Polling-Based Sync

Academic research into browser synchronization highlights a reliance on RESTful polling. Native Firefox Sync protocols prioritize data integrity and server-side resource conservation over immediacy. As noted in studies of modern distributed systems, polling-based updates introduce a "Sync Interval Gap," where state changes on a client are not broadcasted until the next scheduled heartbeat, typically every 10–20 minutes.

Resource Management in Vertical Tab UIs

Existing literature on Sidebery and Tree Style Tab architectures focuses on spatial organization (verticality and nesting) rather than temporal synchronization. While these tools solve the "tab clutter" problem, they operate as isolated client-side instances, failing to address the cognitive load caused by fragmented sessions across multiple devices.

Market Research

  • Firefox Sync (Native)

    Relies on client-side polling and Mozilla's central servers; optimized for backup, not active session mirroring.

  • Sidebery / Tree Style Tab

    Industry leaders in vertical organization but lack cross-device state synchronization for active tabs.

  • Tab Stash / OneTab

    Effective for session archival (parking tabs), but do not support live, bi-directional mirroring between concurrent browser instances.

Innovation Gap

The Shift

Most extensions operate on a Pull Architecture (asking the server for updates). Tabbed-In introduces a Push Architecture via WebSockets.

Low-Latency Mirroring

Using Supabase Realtime to bypass standard sync intervals.

BYOB Data Ownership

A decentralized approach where users host their own sync backend.

System Methodology

1. Event Listening

Background scripts utilize the browser.tabs API to detect creation, updates, and removal events in real-time across the active browser window.

2. WebSocket Broadcast

Changes are pushed to a Supabase Realtime channel. Using a "Broadcast" pattern ensures all peer devices receive state updates in <500ms.

3. UI Rendering

The Sidebery-inspired sidebar reactively updates its DOM, categorizing tabs into "Mirrored" (Global) and "Unique" (Local) sections via CSS Flexbox.
